O Mesmo Destino,
Dois Caminhos

O objetivo foi criar duas aplicações visualmente idênticas, mas estruturalmente únicas.


  • Angular: Organizado, padronizado, modular.
  • React: Flexível, funcional, reativo.

Mapa do Projeto

ANGULAR/src/app/ ├── pages/ (Lógica + HTML + CSS) ├── components/ (Header, Footer) ├── services/ (Auth, Cart) └── db.json (API)
REACT/src/ ├── pages/ (JSX + CSS) ├── components/ (Header, Footer) ├── utils/ (cart.js, config.js) └── db.json (API)

Funcionalidades Principais

Carrinho

Persistência local (LocalStorage) e cálculo automático de totais.

Admin

Área protegida por login. CRUD completo de produtos e usuários.

Imagens

Sistema de fallback: se a foto falhar, uma imagem padrão é exibida.

Checkout

Validação de formulários e resumo detalhado do pedido.

Autenticação

Login e Cadastro funcionais com proteção de rotas.

API JSON

Backend simulado (REST) para dados de produtos e usuários.

Arquitetura: O Motor

Estado Global

Angular: Usa BehaviorSubject nos Services para criar um estado reativo que "avisa" os componentes.


Rotas & Segurança

Angular: Guards (CanActivate) bloqueiam rotas no arquivo de rotas.

Estado Global

React: Usa funções utilitárias (cart.js) e estado local ou Context API simplificado.


Rotas & Segurança

React: Wrappers de Layout verificam a autenticação antes de renderizar a página.

Como Rodar

# Angular npm install npm run api (Porta 3001) npm start (Porta 4200)
# React npm install npx json-server ... npm run dev (Porta 5173)

Conclusão

Este projeto demonstra que, independente da ferramenta (Angular ou React), os fundamentos de arquitetura, organização e UX são o que definem um software de qualidade.
