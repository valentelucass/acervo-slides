Problema resolvido

Não é só cadastrar dados. É controlar risco.

  • Resíduos perigosos precisam de rastreabilidade: origem, tipo, quantidade, transporte e prazo.
  • O manifesto concentra as relações entre gerador, resíduo e transportadora.
  • O diferencial técnico está em validar dados e automatizar decisões que seriam manuais.

Login e permissao

Rotas protegidas por perfil

  • `authGuard` bloqueia paginas internas quando nao existe usuário logado.
  • `adminGuard` limita o cadastro de usuários ao administrador.
  • A sessão fica no `sessionStorage`, suficiente para a proposta acadêmica e para a demo.
export const adminGuard: CanActivateFn = () => {
  const auth = inject(AuthService);
  const router = inject(Router);

  if (auth.isAdmin()) return true;

  router.navigate(['/dashboard']);
  return false;
};

Prazo de descarte calculado pela classe do resíduo

  • Classe I: ate 90 dias.
  • Classe II-A: ate 180 dias.
  • Classe II-B: ate 365 dias.
  • A data aparece no formulário antes de salvar o manifesto.

Services

Lista enriquecida para a tabela

  • O manifesto salva apenas IDs, como em um banco normalizado.
  • `listarComDetalhes` junta as coleções e entrega uma `ManifestoView` pronta para a tela.
  • Isso deixa o HTML simples e concentra a montagem dos dados no service.
return forkJoin({
  manifestos: this.http.get<Manifesto[]>(...),
  geradores: this.geradorService.listar(),
  transportadoras: this.transportadoraService.listar(),
  residuos: this.residuoService.listar(),
}).pipe(
  map(({ manifestos, geradores, transportadoras, residuos }) =>
    manifestos.map((m) => ({
      ...m,
      gerador: geradores.find((g) => g.id === m.geradorId),
      transportadora: transportadoras.find((t) => t.id === m.transportadoraId),
      residuo: residuos.find((r) => r.id === m.residuoId),
    }))
  )
);

Como apresentar

Roteiro de demo em fluxo

  1. Login: entrar como administrador e mostrar o menu de usuários.
  2. CNPJ: tentar cadastrar gerador com CNPJ inválido e mostrar o bloqueio.
  3. MTR: criar manifesto pelo stepper e destacar o prazo calculado.
  4. Tabela: filtrar por status e alterar para transportado/finalizado.
  5. Dashboard: fechar mostrando contadores e alertas de prazo.
